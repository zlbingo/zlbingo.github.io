    python os.walk()的使用方法

    zlbingo 9/14/2021 python用法

    # 1. 语法

    os.walk(top[, topdown=True[, onerror=None[, followlinks=False]]])
    1

    # 2. 参数

    • top: 所要遍历的目录的地址
    • topdown： 字面意思，从上到下，默认为True，先遍历top的目录，然后遍历top的子目录，置为False，反之，下面有示例介绍
    • onerror：用处不大，暂不学习
    • followlinks -- 可选，如果为 True，则会遍历目录下的快捷方式(linux 下是软连接 symbolic link )实际所指的目录(默认关闭)，如果为 False，则优先遍历 top 的子目录。

    # 3. 返回值

    返回的是一个生成器




     

    >>> import os
>>> a =os.walk("/root/test")
>>> print(a)
<generator object walk at 0x7f5eecb49580>
    1
    2
    3
    4

    可通过for循环的方式，遍历这个生成器，每次得到的是一个三元组（dirpath,dirs,files）

    • dirpath所指的是当前正在遍历的这个文件夹的本身的地址
    • dirs 是一个 list ，内容是该文件夹中所有的目录的名字(不包括子目录)
    • files 同样是 list , 内容是该文件夹中所有的文件(不包括子目录)

    • 🕛查看官方说明文档

      dirpath is a string, the path to the directory. dirnames is a list of the names of the subdirectories in dirpath (excluding '.' and '..'). filenames is a list of the names of the non-directory files in dirpath. Note that the names in the lists contain no path components. To get a full path (which begins with top) to a file or directory in dirpath, do os.path.join(dirpath, name). Whether or not the lists are sorted depends on the file system. If a file is removed from or added to the dirpath directory during generating the lists, whether a name for that file be included is unspecified.

    • 🕘后续

      后来在CSDN上发现一篇文章python os.walk 排序_os.walk按什么顺序迭代？ (opens new window)比较接近答案，但感觉像是机翻。

      然后我再stackoverflow反向查找了一下,找到这么一个答案：in-what-order-does-os-walk-iterates-iterate (opens new window)

      答案中有这么一句

      FYI: On Linux/ext3 this is currently the same as ls -U

      然后我就去尝试了一下，

      root@linux:~/test# ls -U
dir_b  dir_a  dir_c
      1
      2

      发现其顺序确实是先dir_b，然后在dir_a

      接着查看了一下-U参数的介绍

      -U do not sort; list entries in directory order

      那么什么是directory order,然后找到了这个答案What is the "directory order" of files in a directory (used by ls -U)? (opens new window)

      这里简单列举了一个答案，大致意思是不同的文件系统为了高性能的对文件进行管理，会有自己的一套排序方式（例如使用红黑树），os.walk直接使用的文件系统的排序方式（这也对应了其官方说明文档中的Whether or not the lists are sorted depends on the file system）
